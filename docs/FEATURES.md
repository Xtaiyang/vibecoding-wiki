## 二、认证与授权模块

### 2.1 用户注册

- **功能描述**：新用户通过邮箱注册账号
- **业务边界**：
  - 仅支持邮箱注册，暂不支持手机号注册
  - 邮箱必须唯一，不可重复注册
  - 密码长度 8-128 位，必须包含字母和数字
- **权限要求**：无（公开访问）
- **交互逻辑**：
  1. 用户填写邮箱、密码、确认密码
  2. 前端校验格式，后端校验邮箱唯一性
  3. 发送验证邮件（异步，不阻塞注册流程）
  4. 注册成功后自动登录，跳转至控制台
- **完成状态**：已完成

### 2.2 用户登录

- **功能描述**：已注册用户通过邮箱+密码登录
- **业务边界**：
  - 支持邮箱+密码登录
  - 密码错误 5 次后锁定账号 30 分钟
  - 登录后签发 Access Token（15分钟）+ Refresh Token（7天）
- **权限要求**：无（公开访问）
- **交互逻辑**：
  1. 用户输入邮箱和密码
  2. 后端校验 credentials，返回 Token
  3. 前端存储 Token，跳转至控制台
  4. Token 过期前自动刷新
- **完成状态**：已完成

### 2.3 密码重置

- **功能描述**：用户通过邮箱验证重置密码
- **业务边界**：
  - 重置链接有效期 1 小时
  - 重置成功后，所有已登录设备强制下线
- **权限要求**：无（公开访问，需验证邮箱）
- **交互逻辑**：
  1. 用户输入注册邮箱
  2. 后端生成重置 Token，发送邮件
  3. 用户点击邮件链接，输入新密码
  4. 更新密码，清除所有 Refresh Token
- **完成状态**：已完成

### 2.4 角色与权限

- **功能描述**：系统支持多角色权限控制
- **业务边界**：
  - 固定角色：OWNER、ACCOUNTANT、CASHIER、VIEWER
  - 角色权限不可自定义，仅系统预设
  - 每个账套内成员有独立角色
- **权限要求**：仅 OWNER 可管理成员角色
- **交互逻辑**：
  1. OWNER 邀请成员（通过邮箱）
  2. 被邀请者接受后，OWNER 分配角色
  3. 各角色按权限访问功能
- **完成状态**：进行中

### 2.5 多设备登录管理

- **功能描述**：查看并管理已登录设备
- **业务边界**：
  - 显示设备类型、登录时间、IP 地址
  - 支持远程登出其他设备
  - 同一账号无设备数量限制
- **权限要求**：需登录
- **完成状态**：待开发

## 三、账套管理模块

### 3.1 创建账套

- **功能描述**：用户创建新的财务账套
- **业务边界**：
  - 每个用户最多创建 10 个账套（免费版 3 个）
  - 账套创建后不可删除，仅可归档
  - 账套信息：名称、启用日期、会计制度（小企业/企业）
- **权限要求**：任何登录用户
- **交互逻辑**：
  1. 填写账套基本信息
  2. 选择会计制度（影响默认科目）
  3. 系统自动生成默认会计科目
  4. 创建者自动成为 OWNER
- **完成状态**：已完成

### 3.2 切换账套

- **功能描述**：在多账套间快速切换
- **业务边界**：
  - 仅显示当前用户有权限的账套
  - 切换后所有数据按新账套加载
  - 最后访问的账套下次登录自动选中
- **权限要求**：需登录，且是账套成员
- **交互逻辑**：
  1. 点击顶部账套选择器
  2. 下拉显示有权限的账套列表
  3. 选择后页面刷新，加载新账套数据
- **完成状态**：已完成

### 3.3 账套成员管理

- **功能描述**：管理账套内的成员和角色
- **业务边界**：
  - OWNER 可添加、移除、修改成员角色
  - 不可移除自己（需先转让 OWNER）
  - 被移除成员的数据保留，仅失去访问权
- **权限要求**：仅 OWNER
- **交互逻辑**：
  1. OWNER 进入账套设置-成员管理
  2. 输入邮箱邀请新成员
  3. 被邀请者邮件确认后加入
  4. OWNER 可修改或移除成员
- **完成状态**：进行中

### 3.4 账套归档

- **功能描述**：将不再使用的账套归档
- **业务边界**：
  - 归档后账套只读，不可新增凭证
  - 归档后可随时恢复
  - 归档不影响报表查看
- **权限要求**：仅 OWNER
- **完成状态**：待开发

## 四、凭证管理模块

### 4.1 凭证录入

- **功能描述**：手动录入会计凭证
- **业务边界**：
  - 凭证日期不可早于账套启用日期
  - 凭证日期不可晚于当前会计期间（可配置）
  - 借贷金额必须平衡
  - 每张凭证至少 2 条明细
- **权限要求**：ACCOUNTANT、CASHIER、OWNER
- **交互逻辑**：
  1. 选择凭证日期、类型（收入/支出/转账）
  2. 填写摘要，添加明细行（科目、借贷金额）
  3. 实时校验借贷平衡
  4. 保存后生成凭证号（按日期顺序）
- **完成状态**：已完成

### 4.2 凭证列表

- **功能描述**：查看、筛选、搜索凭证
- **业务边界**：
  - 支持按日期范围、凭证号、摘要筛选
  - 支持按科目筛选
  - 默认显示当月凭证
  - 分页加载，每页 20 条
- **权限要求**：所有角色（VIEWER 仅查看）
- **交互逻辑**：
  1. 进入凭证管理页面，默认加载当月
  2. 顶部筛选栏支持日期、凭证号、摘要
  3. 点击凭证行展开明细
  4. 支持导出 Excel
- **完成状态**：已完成

### 4.3 凭证编辑与删除

- **功能描述**：修改或删除已保存凭证
- **业务边界**：
  - 仅当月凭证可编辑（已结账月份不可修改）
  - 删除凭证后凭证号不重新编排（留空号）
  - 已审核凭证需取消审核后才能编辑
- **权限要求**：ACCOUNTANT、OWNER（CASHIER 只能编辑自己创建的）
- **交互逻辑**：
  1. 在列表中点击编辑
  2. 校验是否可编辑（月份、审核状态）
  3. 编辑后重新校验借贷平衡
  4. 删除需二次确认
- **完成状态**：进行中

### 4.4 凭证审核

- **功能描述**：会计主管审核凭证
- **业务边界**：
  - 审核后凭证不可编辑
  - 审核人与制单人不能为同一人（可选配置）
  - 支持批量审核
- **权限要求**：ACCOUNTANT、OWNER
- **交互逻辑**：
  1. 在列表中选择待审核凭证
  2. 点击审核按钮
  3. 审核通过后状态变为"已审核"
  4. 支持取消审核（需有权限）
- **完成状态**：待开发

### 4.5 凭证附件

- **功能描述**：上传凭证相关附件（发票扫描件等）
- **业务边界**：
  - 支持 PDF、JPG、PNG 格式
  - 单个文件最大 10MB
  - 每张凭证最多 5 个附件
- **权限要求**：ACCOUNTANT、CASHIER、OWNER
- **交互逻辑**：
  1. 在凭证录入/编辑页点击上传
  2. 前端校验格式和大小
  3. 上传到 MinIO，返回 URL
  4. 附件与凭证关联存储
- **完成状态**：待开发

### 4.6 智能凭证生成

- **功能描述**：AI 根据描述自动生成凭证分录
- **业务边界**：
  - 基于 OpenAI API，需用户确认后保存
  - 生成结果仅供参考，不自动保存
  - 复杂业务场景可能不准确
- **权限要求**：ACCOUNTANT、OWNER
- **交互逻辑**：
  1. 用户输入交易描述和金额
  2. 调用 AI 服务生成建议分录
  3. 用户确认或修改后保存
  4. 不满意的建议可重新生成
- **完成状态**：待开发

## 五、发票管理模块

### 5.1 发票录入

- **功能描述**：录入发票信息
- **业务边界**：
  - 支持增值税专用发票、普通发票
  - 发票号码唯一性校验
  - 发票日期不可晚于当前日期
- **权限要求**：ACCOUNTANT、CASHIER、OWNER
- **完成状态**：待开发

### 5.2 发票识别

- **功能描述**：上传发票图片，AI 自动识别信息
- **业务边界**：
  - 支持 JPG、PNG、PDF 格式
  - 识别结果需用户确认
  - 识别失败时提示手动录入
- **权限要求**：ACCOUNTANT、CASHIER、OWNER
- **完成状态**：待开发

### 5.3 发票查验

- **功能描述**：对接税务局接口查验发票真伪
- **业务边界**：
  - 仅支持查验已录入的发票
  - 查验结果缓存 7 天
  - 每日查验次数限制 100 次
- **权限要求**：ACCOUNTANT、OWNER
- **完成状态**：待开发

### 5.4 发票与凭证关联

- **功能描述**：将发票与凭证关联
- **业务边界**：
  - 一张发票可关联多张凭证
  - 关联后发票状态变为"已记账"
  - 取消关联后发票恢复"未记账"
- **权限要求**：ACCOUNTANT、OWNER
- **完成状态**：待开发

### 5.5 发票统计

- **功能描述**：按税率、类型、期间统计发票
- **业务边界**：
  - 统计结果可导出 Excel
  - 支持进项/销项分别统计
  - 税额自动汇总
- **权限要求**：所有角色
- **完成状态**：待开发

## 六、报表中心模块

### 6.1 资产负债表

- **功能描述**：生成资产负债表
- **业务边界**：
  - 支持按月、按季度、按年生成
  - 数据来源于已审核凭证
  - 期初余额需手动录入或继承上月
- **权限要求**：所有角色
- **完成状态**：待开发

### 6.2 利润表

- **功能描述**：生成利润表（损益表）
- **业务边界**：
  - 支持按月、按季度、按年生成
  - 支持同比、环比分析
  - 数据来源于已审核凭证
- **权限要求**：所有角色
- **完成状态**：待开发

### 6.3 现金流量表

- **功能描述**：生成现金流量表
- **业务边界**：
  - 支持直接法和间接法
  - 需指定现金及现金等价物科目
  - 数据来源于已审核凭证
- **权限要求**：所有角色
- **完成状态**：待开发

### 6.4 科目余额表

- **功能描述**：按科目展示期初、本期发生、期末余额
- **业务边界**：
  - 支持按期间筛选
  - 支持按科目层级展开/收起
  - 点击科目可查看明细账
- **权限要求**：所有角色
- **完成状态**：待开发

## 七、会计科目模块

### 7.1 科目管理

- **功能描述**：管理会计科目体系
- **业务边界**：
  - 系统预设标准科目（根据会计制度）
  - 支持新增、修改、停用自定义科目
  - 系统预设科目不可删除
  - 已有余额的科目不可删除
- **权限要求**：OWNER、ACCOUNTANT
- **完成状态**：待开发

### 7.2 期初余额录入

- **功能描述**：录入科目期初余额
- **业务边界**：
  - 仅在账套启用初期或新年度可录入
  - 借贷总额必须平衡
  - 录入后影响报表期初数
- **权限要求**：OWNER、ACCOUNTANT
- **完成状态**：待开发

### 7.3 辅助核算

- **功能描述**：为科目启用辅助核算（客户、供应商、项目等）
- **业务边界**：
  - 仅自定义科目可启用辅助核算
  - 辅助核算类型：客户、供应商、部门、项目、员工
  - 启用后凭证录入需填写辅助信息
- **权限要求**：OWNER、ACCOUNTANT
- **完成状态**：待开发

## 八、系统设置模块

### 8.1 账套信息设置

- **功能描述**：修改账套基本信息
- **业务边界**：
  - 账套名称、启用日期可修改
  - 会计制度不可修改（影响科目体系）
- **权限要求**：OWNER
- **完成状态**：待开发

### 8.2 结账与反结账

- **功能描述**：月度/年度结账
- **业务边界**：
  - 结账前所有凭证必须已审核
  - 结账后该月凭证不可编辑
  - 支持反结账（OWNER 权限）
- **权限要求**：OWNER、ACCOUNTANT
- **完成状态**：待开发

### 8.3 数据备份

- **功能描述**：手动或自动备份账套数据
- **业务边界**：
  - 支持导出为 Excel/JSON
  - 自动备份每日一次，保留 30 天
  - 备份文件加密存储
- **权限要求**：OWNER
- **完成状态**：待开发

### 8.4 操作日志

- **功能描述**：查看账套操作记录
- **业务边界**：
  - 记录增删改操作（谁、何时、做了什么）
  - 日志不可删除、不可修改
  - 保留 1 年
- **权限要求**：OWNER、ACCOUNTANT（查看）
- **完成状态**：待开发

## 九、AI 助手模块

### 9.1 智能发票识别

- **功能描述**：上传发票图片，AI 提取关键信息
- **业务边界**：
  - 支持增值税发票、普通发票
  - 识别准确率目标 95%+
  - 识别结果需人工确认
- **权限要求**：ACCOUNTANT、CASHIER、OWNER
- **完成状态**：待开发

### 9.2 智能凭证建议

- **功能描述**：根据交易描述自动生成会计分录
- **业务边界**：
  - 基于历史凭证学习
  - 生成结果仅供参考
  - 复杂业务可能需人工调整
- **权限要求**：ACCOUNTANT、OWNER
- **完成状态**：待开发

### 9.3 财务问答

- **功能描述**：AI 解答财务、税务问题
- **业务边界**：
  - 基于通用财税知识回答
  - 不提供具体税务筹划建议（法律风险）
  - 回答仅供参考，不构成专业意见
- **权限要求**：所有角色
- **完成状态**：待开发

## 十、功能边界总览

| 边界类型 | 规则 |
|---------|------|
| 数据范围 | 用户只能访问自己有权限的账套数据 |
| 时间范围 | 凭证日期不可早于账套启用日期，不可晚于当前期间（可配置） |
| 修改限制 | 已结账期间的凭证不可编辑 |
| 审核限制 | 审核人与制单人建议不同（可配置） |
| 删除限制 | 账套不可删除，仅可归档 |
| 角色限制 | OWNER 拥有所有权限，其他角色按预设权限访问 |
| AI 边界 | AI 生成内容仅供参考，不自动保存，需人工确认 |
