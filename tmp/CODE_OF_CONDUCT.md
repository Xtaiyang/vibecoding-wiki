# 贡献者行为准则

> 本文档定义了项目贡献者公约、行为规范与承诺说明。所有参与项目开发和维护的人员均需遵守。

## 一、我们的承诺

作为项目成员、贡献者和维护者，我们承诺：

- 为所有人提供友好、安全和受欢迎的体验，无论年龄、体型、残障、种族、性别认同和表达、经验水平、国籍、个人外貌、种族、宗教或性取向如何
- 以建设性和专业的方式对待彼此
- 尊重不同的观点和经验
- 优雅地接受建设性的批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

## 二、我们的标准

### 2.1 积极行为示例

- 使用友好和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性的批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 2.2 不可接受的行为

- 使用带有性暗示的语言或图像，以及不受欢迎的性关注或性挑逗
- 发表挑衅、侮辱或贬损的评论，进行人身攻击或政治攻击
- 公开或私下骚扰他人
- 未经明确许可，发布他人的私人信息（如物理地址或电子邮件地址）
- 其他在专业环境中被合理认为不适当的行为

## 三、责任范围

本行为准则适用于所有项目空间，包括：

- 代码仓库（GitHub Issues、Pull Requests）
- 项目文档和评论
- 项目相关的社交媒体和公开活动
- 项目维护者的私人通信（如果涉及项目事务）

## 四、执行机制

### 4.1 举报不当行为

如果您遇到或目睹不可接受的行为，请通过以下方式举报：

- 发送邮件至：conduct@finance-platform.com
- 向项目维护者私信

### 4.2 处理流程

1. **接收举报**：维护团队收到举报后 24 小时内确认
2. **调查核实**：维护团队调查事件，可能联系相关方
3. **做出决定**：根据严重程度，决定采取的措施
4. **执行措施**：通知相关方并执行决定
5. **记录存档**：记录事件和处理结果

### 4.3 处罚措施

| 严重程度 | 示例 | 措施 |
|---------|------|------|
| 轻微 | 不专业的语言、轻微的冒犯 | 口头或书面警告 |
| 中等 | 持续的不当行为、骚扰 | 临时封禁（7-30 天） |
| 严重 | 严重的骚扰、歧视、威胁 | 永久封禁 |

## 五、贡献规范

### 5.1 提交代码

- 所有代码提交必须通过 Pull Request
- PR 描述必须清晰说明变更内容和原因
- 必须通过 CI 检查（lint、typecheck、test）
- 至少一名维护者 Review 通过后方可合并

### 5.2 代码审查

- 审查时保持尊重和专业
- 关注代码质量而非个人风格
- 提供具体的改进建议
- 及时响应审查意见

### 5.3 Issue 规范

- 提交 Issue 前搜索是否已存在
- 使用提供的 Issue 模板
- 提供足够的信息（复现步骤、环境、错误信息）
- 对 Issue 进行适当的分类和标签

## 六、维护者承诺

项目维护者承诺：

- 公正、一致地执行本行为准则
- 保护举报者的隐私
- 及时响应举报和关切
- 定期审查和更新本准则
- 以身作则，遵守更高的标准

## 七、适用范围

本行为准则适用于：

- 所有项目贡献者（代码、文档、设计、测试等）
- 项目维护者和核心团队
- 社区成员和用户
- 项目相关的所有活动和空间

## 八、准则更新

本行为准则将根据社区反馈和项目发展定期更新。更新将经过社区讨论，并由维护者最终确认。

- **当前版本**：v1.0.0

- **最后更新**：2026-06-26
